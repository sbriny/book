## 第一章 功能概述

### 1.1 功能定位

Dock 是桌面系统的核心导航组件，固定在屏幕底部居中，提供应用快速启动、运行状态感知、窗口聚焦/恢复等能力。

### 1.2 核心能力

| 能力 | 说明 |
| :-- | :-- |
| 应用启动 | 点击未运行应用图标启动新窗口 |
| 窗口聚焦 | 点击已运行应用图标聚焦其窗口 |
| 最小化恢复 | 点击已最小化图标恢复窗口 |
| 多实例管理 | 右键菜单选择具体窗口或新开实例 |
| 运行指示器 | 运行中显示白色小圆点，最小化另有样式 |
| 多实例徽章 | 实例数 \> 1 时右上角显示数字徽章 |
| 悬浮提示 | hover 显示应用名、运行状态、上次打开时间 |
| 入场动画 | 从中心向两侧凝结，配合容器背景闪现 |

### 1.3 技术栈

- **React 18** \+ Hooks
- **framer-motion**：图标入场动画、交互动效
- **WindowManagerContext**：窗口状态数据源
- **AppRegistry**：应用列表数据源

## 第二章 架构设计

### 2.1 整体结构

```text
Dock (状态中枢)
│
├── tooltip (fixed 定位，在 dock-wrapper 外独立渲染)
│
├── dock-wrapper (mix-blend-mode: difference)
│   └── dock-container (入场闪现背景)
│       └── DockIcon × N
│           ├── motion.button (dock-icon-btn)
│           │   ├── IconComponent (dock-icon-svg)
│           │   ├── dock-indicator (运行中)
│           │   ├── dock-indicator-minimized (最小化)
│           │   └── dock-badge (多实例)
│           └── (tooltip 数据向上传，不在此渲染)
│
└── WindowInstanceMenu (AnimatePresence 包裹)
```

### 2.2 数据流

```text
AppRegistry.getDockApps()
        │
        ▼
   dockApps 列表
        │
        ▼
WindowManagerContext
(windows / focus / restoreWindow)
        │
        ▼
    Dock 计算层
    ┌─────────────────────┐
    │ isAppRunning        │
    │ isAppMinimized      │
    │ getAppWindows       │
    │ getLastFocusedAt    │
    └─────────────────────┘
        │
        ▼
    DockIcon (只接收 props，无内部业务逻辑)
        │
   onMouseEnter → setTooltip (上传到 Dock)
```

### 2.3 关键机制

#### tooltip 渲染位置

tooltip 必须渲染在 `.dock-wrapper` **外部**，与其平级。

原因：`.dock-wrapper` 设有 `mix-blend-mode: difference`，会创建新的 stacking context，子元素的颜色和混合模式无法独立控制，tooltip 渲染在内部会受到 difference 影响导致颜色异常。

```text
return (
    <>
        {tooltip && <div className="dock-tooltip" style={tooltipStyle} />}
        <div className="dock-wrapper">
            ...
        </div>
    </>
);
```

tooltip 定位通过 `getBoundingClientRect()` 获取 icon 的屏幕坐标，配合 `position: fixed` 实现精确定位，不依赖父元素的定位上下文。

#### mix-blend-mode: difference

`.dock-wrapper` 使用 `mix-blend-mode: difference`，让图标颜色自动适应壁纸背景：

- 深色壁纸下图标显示为浅色
- 浅色壁纸下图标显示为深色
- 彩色壁纸下产生补色，始终保持可辨识度

配合 `.dock-icon-svg` 的双层 `drop-shadow`（白色光晕 \+ 黑色扩散晕），在复杂图片背景下保证边缘清晰：

```text
.dock-icon-svg {
    filter:
        drop-shadow(0 0 1.3px rgba(255,255,255,0.8))
        drop-shadow(0 0 1px rgba(0,0,0,0.5));
}
```

#### 入场动画

采用 visionOS 风格"从中心向两侧凝结"方案：

- 动画单元为 `.dock-icon-btn`（`motion.button`）
- 每个图标的 delay 由其与中心位置的距离决定：`dist * 0.08s`
- 动画参数：`opacity 0→1`、`scale 0.75→1`、`filter blur(12px)→0`
- 物理参数：`spring stiffness:200 damping:22 mass:0.8`

容器背景闪现与图标动画同步：动画开始时 `.dock-container` 出现半透明背景，最后一个图标动画结束后背景消失。

```text
delay 计算：
index=0  dist=3  delay=0.24s
index=1  dist=2  delay=0.16s
index=2  dist=1  delay=0.08s
index=3  dist=0  delay=0s      ← 中心，最先出现
index=4  dist=1  delay=0.08s
index=5  dist=2  delay=0.16s
index=6  dist=3  delay=0.24s
```

容器背景消失时机：`maxDelay * 1000 + 800ms`（800 为 spring 动画约定结束时长）。

#### 多实例处理

```text
getAppWindows(appId)
    │
    ├── length === 0  → launchApp()
    ├── length === 1  → isMinimized ? restoreWindow : focus
    └── length > 1   → 操作最近聚焦的窗口
                        右键菜单可选择具体实例
```

## 第三章 样式说明

### 3.1 层级关系

| 元素 | z-index | 说明 |
| :-- | :-- | :-- |
| `.dock-wrapper` | 50 | dock 主体 |
| `.dock-tooltip` | 9999 | 悬浮提示，fixed 定位需要高层级 |
| `WindowInstanceMenu` | （由自身决定） | 右键菜单 |

### 3.2 图标颜色方案

| 方式 | 作用 |
| :-- | :-- |
| `mix-blend-mode: difference` | 图标主色随壁纸自动反色 |
| `drop-shadow` 白色层 | 深色背景下保证边缘可见 |
| `drop-shadow` 黑色层 | 浅色背景下保证边缘可见 |
| `stroke` 描边 | 图标线条轻微加粗，提升辨识度 |

### 3.3 状态样式

| 状态 | 表现 |
| :-- | :-- |
| 默认 | 无背景 |
| hover | `background: rgba(20,20,20,0.2)` \+ `scale(0.95)` |
| active（运行中） | `background: rgba(20,20,20,0.7)` \+ 底部绿色圆点 |
| minimized | `opacity: 0.6` \+ 底部白色小圆点 |
| 多实例 | 右上角蓝色数字徽章 |

### 3.4 tooltip 样式

tooltip 使用毛玻璃效果：`background: rgba(20,20,20,0.2)` \+ `backdrop-filter: blur(8px)`。

箭头不使用 `border` 三角形（颜色无法与毛玻璃背景保持一致），也不使用伪元素（`backdrop-filter` 在伪元素上大多数浏览器不支持），当前方案为无箭头设计，通过位置关系足以建立 tooltip 与 icon 的关联。
